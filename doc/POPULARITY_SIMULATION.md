# 发帖热度变化模拟

公式：`总热度 = (收藏×Wc + 点赞×Wl + 浏览×Wv) × 互动衰减`

- **配比**：`SEARCH_WEIGHT_COLLECT`、`SEARCH_WEIGHT_LIKE`、`SEARCH_WEIGHT_VIEW`（默认 10、5、1）
- **互动衰减** = `1 / (1 + 距今天数 / SEARCH_INTERACTION_DECAY_DAYS)`（默认 90 天半衰期）

## 场景一：普通帖（收藏 5、点赞 20、浏览 300）

互动分基数 = 50 + 100 + 300 = **450**

| 距今天数  | 互动衰减系数 | 有效互动分（总热度） |
|-------|--------|------------|
| 0 天   | 100%   | **450.0**  |
| 1 天   | 99%    | 445.1      |
| 7 天   | 93%    | 417.6      |
| 14 天  | 86%    | 388.2      |
| 30 天  | 75%    | 337.5      |
| 90 天  | 50%    | 225.0      |
| 180 天 | 33%    | 150.0      |
| 365 天 | 20%    | 89.0       |

## 场景二：冷门帖（收藏 1、点赞 5、浏览 50）

互动分基数 = 10 + 25 + 50 = **85**

| 距今天数 | 总热度  |
|------|------|
| 0 天  | 87.0 |
| 7 天  | 79.2 |
| 30 天 | 64.1 |
| 90 天 | 42.5 |

## 场景三：热门帖（收藏 20、点赞 100、浏览 2000）

互动分基数 = 200 + 500 + 2000 = **2700**

| 距今天数  | 总热度    |
|-------|--------|
| 0 天   | 2702.0 |
| 7 天   | 2508.0 |
| 30 天  | 2027.6 |
| 90 天  | 1351.0 |
| 365 天 | 535.0  |

## 结论

1. **随时间衰减**：90 天时互动分约为发布时的 50%，365 天约为 20%。
2. **冷门 vs 热门**：高互动帖在早期排名靠前，但随时间衰减后，新旧帖差距会缩小，有利于新内容曝光。

运行 `go run scripts/simulate_popularity.go` 可自行调整参数重新模拟。
