# 订单与聊天（平台不经手资金）

本系统**不处理任何资金**，买卖双方线下沟通、扫码付款。**流程约定**：沟通完毕 → **买方先表示已付款并下单** → **卖方确认收款
** → 派送/自提 → **买方确认收货** → 交易结束。

## 商品类别 `goods_type`（商品表）

| 值 | 含义             | 订单履约                                                                                                                      |
|---|----------------|---------------------------------------------------------------------------------------------------------------------------|
| 1 | **送货上门**       | 商品 **商品地址**（`goods_addr`）可作为默认 **卖方发货地址**（`sender_addr`）。卖方确认收款后 **正在派送** → 卖方 `confirm-delivery` → 待买方确认收货；买卖地址齐全时可算步行距离 |
| 2 | **自提**         | 商品 **商品地址** 为约定提货点。买方下单后同上：**买方已付款下单 → 卖方确认收款** → **待买方自提**；买方 **确认收货** 表示已提货完成；**不计算**送货距离                               |
| 3 | **在线商品**（会员卡等） | **无实体派送**；卖方确认收款后 **直接进入待买方确认收货**（不经过 `confirm-delivery`）；买方确认收货后扣库存                                                      |

## 订单状态 `order_status`

| 值 | 含义                                         |
|---|--------------------------------------------|
| 1 | **待买方付款下单**：可聊天；买方未「已付款下单」前可磋商             |
| 2 | **待卖方确认收款**：买方已表示已付款并下单（`buyer_agreed_at`） |
| 3 | **履约中**：送货上门=正在派送；自提=待买方自提                 |
| 4 | **待买方确认收货**：已送达（实体）或在线/待买方最终确认             |
| 5 | **已完成**：买方确认收货，**库存 -1**                   |
| 6 | **已取消**                                    |

## 典型 API 顺序（前端）

### 送货上门

1. `POST /api/v1/orders` — `{ "goods_id", "receiver_addr"?, "sender_addr"?, "buyer_claim_paid"? }`
    - 默认进入状态 **1**；可选 `buyer_claim_paid: true` 创建并直接进入状态 **2**（待卖方确认收款）。
2. `GET/POST /api/v1/orders/:id/messages` — 聊天（未完成前）
3. `POST /api/v1/orders/:id/buyer-claim-paid` — 买方 **已付款下单** → 状态 **2**
4. `POST /api/v1/orders/:id/seller-confirm-payment` — 卖方 **确认收款** → 状态 **3**（正在派送）
5. `PUT /api/v1/orders/:id` — 卖方可填 `sender_addr`（状态 1～3）
6. `POST /api/v1/orders/:id/confirm-delivery` — 卖方送达（可选 `delivery_images`）→ 状态 **4**
7. `POST /api/v1/orders/:id/confirm-receipt` — 买方 → 状态 **5**

### 自提

1～2 同上门；**第 3～4 步** 同 **买方已付款下单**、**卖方确认收款** → 状态 **3**（待买方自提）；**无** `confirm-delivery`；买方在状态
**3** 调用 `confirm-receipt` → 状态 **5**。

### 在线商品

1～2 同上门；**第 3～4 步** 卖方确认收款后 **直接** 状态 **4**（待买方确认收货）；**无** `confirm-delivery`；买方
`confirm-receipt` → 状态 **5**。

### 取消

`POST /api/v1/orders/:id/cancel` — 状态 **1、2、3** 可取消 → **6**。

图片请先走 `POST /api/v1/oss/...` 上传，再将返回 URL 写入消息或确认接口。
